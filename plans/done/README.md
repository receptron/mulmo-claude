# Archived plans — shipped work

Plans in this directory describe features and fixes that have been
merged into `main`. They are kept for historical reference (threat
models, design rationale, "why did we choose X" answers) but are not
active work items.

When a new plan lands in `plans/` and its corresponding PR merges,
move it here with `git mv plans/<name>.md plans/done/<name>.md` and
update any `// See plans/<name>.md` comments in code to
`plans/done/<name>.md`.

Active plans live one level up at `plans/`.
